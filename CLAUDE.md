# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

---

Игра-лабиринт для детей 9–13 лет (Phaser 3, чистые ES-модули без сборщика, PWA).
Главный игрок и тестировщик — **Лиана**. **Всё общение и комментарии в коде —
на русском.** Ниже — то, что не восстановить из самого кода: почему сделано
именно так, где грабли и как тут вообще запускать и тестировать.

**Сейчас в игре:** 10 уровней, монеты (за прохождение +5 и разбросаны по карте),
магазин со скинами и суперспособностями, экран выбора уровня (он же фарм),
прогресс в localStorage. 8 игроков (4 беспл. + 4 платных), 7 врагов (2 беспл. +
5 платных), 2 суперспособности (косточка-приманка, щит), батуты на карте
(временный буст скорости, не расходник).

## Команды

Нет сборщика, нет `package.json`, нет lint/test-раннера — это открытые ES-модули,
которые браузер грузит напрямую. «Запустить проект» = поднять статический сервер
и открыть в браузере; «протестировать» = прогнать игру через встроенную в
браузерную панель JS-консоль (см. раздел «Тестирование» ниже).

**Запуск локально** (открыть `index.html` по `file://` нельзя — браузер запретит
ES-модули и Service Worker):

```
python -m http.server 8080     # из корня проекта
# затем http://localhost:8080/?nosw
```

⚠️ Если `node`/`python` «не найдены» в PowerShell — не верь, проверь полным путём
(`C:\Program Files\nodejs\node.exe`) или через реестр
(`[Environment]::GetEnvironmentVariable("Path","Machine")`): PATH мог обновиться
уже после старта текущей сессии.

Запасной сервер без зависимостей (умеет ещё принимать скриншоты POST-ом на
`/upload`, сохраняет в `tools/_shots/`):

```
powershell -ExecutionPolicy Bypass -File tools\serve.ps1
```

**Всегда открывай с `?nosw`.** Service Worker кеширует всё и будет отдавать
старый код после правок — `?nosw` снимает SW и чистит кеш при загрузке страницы.

**Нарезка спрайтов** (из контактных листов в `images/` в `assets/sprites/`):
`tools/cut_cats.ps1` / `tools/cut_dogs.ps1` (несколько персонажей на лист) или
`tools/cut_new.ps1` (один персонаж на лист, раскладка «6 бега + 2 стойки + 2
позы»). `tools/preview.ps1 <лист> <минПлощадь> <выход>` рисует найденные рамки
с номерами — так подбираются координаты кадров перед вписыванием в скрипт.

## Архитектура

```
src/config/   assets.js (картинки + PALETTE), levels.js (10 уровней + баланс),
              cats.js (скины котиков), dogs.js (породы собак),
              abilities.js (косточка/щит: цена, kind active/passive),
              progress.js (СОХРАНЕНИЕ: монеты/скины/прогресс/инвентарь, 1 ключ localStorage)
src/scenes/   Boot → Menu → LevelSelect → Game + UI; Shop (магазин/кастомизация)
src/systems/  MazeGenerator, Pathfinder (BFS), Joystick (плавающий, левая половина)
src/entities/ Cat (движение + 5 idle + hasShield), Dog (погоня + distract/freeze)
tools/        нарезка спрайтов из контактных листов + локальный сервер
```

**Навигация сцен:** Menu (выбор котика, монеты, магазин) → LevelSelect (8 узлов,
гейт по `progress.maxLevel`, фарм) → Game (+UI HUD отдельной сценой, чтобы не
ездила с камерой). Shop открывается из Menu, LevelSelect и паузы. Прохождение
уровня ведёт на следующий; последний — обратно в LevelSelect.

**Прогресс — единая точка правды.** Всё чтение/запись через `src/config/progress.js`
(один JSON в `localStorage['cat_maze_save']`): монеты, разблокированные скины,
включённые породы врагов, инвентарь способностей, `maxLevel`. Модуль **кеширует
`state` в памяти** — см. ловушку тестирования ниже.

**Data-driven конфиги.** Котики, породы собак и способности — плоские массивы
`{id, name, price, ...}` в `cats.js`/`dogs.js`/`abilities.js`. Добавление нового
элемента = новая строка + спрайты, магазин и сохранение подхватывают сами
(`price: 0` = бесплатный и открыт всегда). `abilities.js` дополнительно несёт
`kind: 'active' | 'passive'` — определяет, рисует ли `UIScene` кнопку активации.

**Лабиринт:** recursive backtracker (гарантирует достижимость выхода) + частичное
«плетение» тупиков в петли (`braid`), иначе от стаи собак некуда отступать.
**Собака:** ищет путь через BFS (`Pathfinder.js`), пересчитывает раз в 500 мс.
Погоня (15 сек) идёт **со встречи**, а не со спавна; таймер стаи общий — все
уходят и возвращаются волной. Полная механика спавна/погони/суперспособностей —
см. таблицу решений ниже, там же обоснование конкретных чисел.

**Ассеты:** все пути к картинкам — только в `src/config/assets.js`. Персонаж на
исходной картинке смотрит **вправо** (влево игра отражает сама). Размер на экране
задают константы (`CAT_SIZE`/`DOG_SIZE`/`SPRITE_SIZE`), не размер файла.

## Решения и почему именно так (не менять не подумав)

| Решение | Почему |
|---|---|
| Phaser 3 локально в `vendor/` | С CDN не работает офлайн, а офлайн — требование PWA |
| Вид сверху с наклоном, **не изометрия** | Ромбы хуже читаются на узком экране телефона |
| Стены рисованные, не тайлсет | `images/Lab_walls.png` — **изометрический**, на квадратную сетку не ложится |
| Генератор: recursive backtracker | Гарантирует, что выход достижим всегда (проверено на 1000 лабиринтов) |
| «Плетение» тупиков (`braid`) | Чистый DFS = одни тупики. С 3 собаками некуда отступать — нечестно |
| Поиск пути: BFS, не A* | Карта ≤35×35. BFS даёт кратчайший путь без эвристик; A* тут лишний код |
| Пересчёт маршрута раз в 500 мс | Дешевле и выглядит живее — собака будто думает |
| Собака медленнее бегущего котика (170) | Иначе убежать нельзя в принципе. Доля задаётся уровнем: 0.7/0.8/0.9 → 119/136/153 |
| 15 сек собаки идут **со встречи**, а не со спавна | Спавн стал случайным по всей карте, но за 15 сек собака пробегает ~40 тайлов при медиане 65–88 и максимуме 326 — далёкая собака не дошла бы никогда |
| Таймер стаи общий: истёк у одной — уходят ВСЕ | С личными таймерами собаки исчезали/появлялись по кругу без передышки. Сигнал `vanish('chase')` эмитится ДО анимации растворения, иначе стая уходит вразнобой. Возвращаются волной через `respawnGap` |
| Кнопка «БЕГ» удалена, бег теперь ЕДИНСТВЕННАЯ скорость | Сначала была переключателем (неудобно держать её и джойстик одной рукой), потом убрали вовсе — котик всегда на `SPEED.catRun`. `catWalk` и весь `input.running` (Cat.js, Joystick.js) выпилены как мёртвый код. Освободившееся место в HUD занял бар способностей |
| Порода собаки случайна при каждом спавне | Стая разношёрстная, встречи разнообразнее. Породы — в `src/config/dogs.js`, добавление новой описано там же |
| Пауза ⏸ (правый верх) → Продолжить / Заново / В меню | «Заново» = restart текущего уровня (лабиринт будет новый — так устроен restart) |
| `roundPixels: true` в конфиге рендера | Котик «размазывался» на телефоне: при дробном зуме камеры (~0.87) спрайт рисовался между пикселями |
| Пол рисуется одним `Graphics` | Тысяча спрайтов = 20 fps на 5-м уровне вместо 60 |
| Весь прогресс — 1 JSON в localStorage (`cat_maze_save`) | В подобных играх сохранение = локальное хранилище браузера, сервер не нужен. Всё через `src/config/progress.js`, формат в одном месте. Миграция старого `cat_maze_skin` → `selectedCat` |
| Монеты банкуются ТОЛЬКО при победе | Иначе фарм = ферма смертей: зашёл, собрал, умер, повтор. Собранное в забеге живёт в `GameScene.collected`, коммитится в `_win`, при `_lose` сгорает |
| Скины data-driven: `price` в cats.js/dogs.js | `price:0` = бесплатный, открыт всегда (форс-разблокировка в progress). Новый платный скин — строка с `price>0` + картинки; магазин подхватит |
| Игрок сам включает породы врагов (enabledDogs) | Купил породу — включена; `randomEnemyBreed` тянет только из enabled; минимум одна активна всегда |
| После финального уровня → LevelSelect, не Menu | Чтобы фарм был под рукой: перепройти любой открытый уровень |
| Способности — стек зарядов (`progress.items`), НЕ разовая разблокировка | Купил дважды — 2 использования. Отдельный generic API (`getItemCount/buyItemCharge/useItemCharge`) от `unlockedCats`-паттерна: там «куплено раз и навсегда», тут расходуется |
| Щит — БЕЗ кнопки, применяется автоматически | Куплен в магазине → заряд списывается и аура включается на старте СЛЕДУЮЩЕГО уровня. Найден на карте → применяется мгновенно при подборе. Одна активная кнопка (для косточки) проще для ребёнка, чем две |
| Радиус косточки — BFS-шаги по лабиринту, не по прямой | `distanceMap` от тайла косточки — та же утилита, что уже считает дальность спавна собак. По прямой через стену собака бы «слышала» косточку нечестно |
| `BONE_DISTRACT_MS` считается от АКТИВАЦИИ, а не от прихода к косточке | Отслеживать приход каждой собаки отдельно — лишняя сложность. Общий таймер на 6 сек (запас на подход) визуально неотличим от «5 сек стоят у косточки» |
| Замороженная собака (`frozenUntil`) не ловит НИКОГДА, не только пока щит активен | Собака стоит на месте, физически остаётся в зоне касания, и на следующем кадре (щит уже снят) засчитывала бы вторую нелегитимную поимку тем же прыжком. Проверка `dog.frozenUntil > time` в `_onDogTouch` — раньше проверки `hasShield` |
| Магазин: вертикальный скролл, а не ужатие карточек | После добавления способностей контент (8 котиков+7 собак+2 способности) перестал влезать по высоте. Сжимать `size` дальше — тупик: при следующем добавлении сломается опять, а мелкие карточки хуже тапать пальцем (аудитория 9–13 лет). `scrollContent`-контейнер под геометрической маской + драг (`pointerdown/move`, клампится в `[0, maxScroll]`) + колесо мыши для десктопа |
| Тап по карточке магазина — на `pointerup`, не `pointerdown`, с порогом 10px | Раньше `pointerdown` стрелял мгновенно при касании — драг для скролла по карточке засчитывался бы как покупка/выбор. `dragDistance` меряется от `pointerdown` до `pointerup`; если меньше порога — это тап, иначе скролл |
| Батут — буст скорости через `cat.boostUntil` (таймстамп), не булев флаг типа `hasShield` | Эффект угасает сам по времени, а не по отдельному событию — тот же паттерн, что уже решён для `dog.frozenUntil`; третий вид "временного состояния" не нужен |
| Батут не исчезает после использования, в отличие от монет/щита | Это элемент карты, а не расходуемый предмет. Кулдаун `readyAt` на самом объекте батута защищает от спама буста, пока котик стоит на месте |
| На батуте (пока горит буст) котик перепрыгивает собаку без урона | «Прыжок» = тот же `cat.boostUntil`, что и ускорение: проверка `cat.boostUntil > time` в `_onDogTouch` РАНЬШЕ `isSafe`/`hasShield`. Пока летит — золотой тинт (ставится каждый кадр в `update`, т.к. `Cat.update` трогает scale, но не tint), чтобы иммунитет читался, а не выглядел багом |
| Squash-tween при прыжке — на самом батуте, не на котике | `Cat.update()` каждый кадр бега безусловно ставит `setScale(baseScale, baseScale+wobble)` (пружинящий бег) — твин на scale котика был бы затёрт на следующем же кадре. Батут анимирует себя сам, буст скорости котика виден по факту движения |
| Раскладка LevelSelect ужалась (`cell`/`gap`/`startY`), не добавлен скролл | Один лишний ряд (10 уровней вместо 8) под фиксированный экран — не открытый рост контента, как было в магазине. Скролл был бы избыточен для статичной сетки из 5 рядов |

**Правило читаемости:** стена обязана быть заметно **темнее** пола. Первая версия
провалилась именно на этом — светлый пол и светлая стена сливались, было непонятно,
где проход. Стены собираются в блоки тёмным контуром + боковая грань снизу (эффект 2.5D).

## Тестирование: браузерная панель — фоновая вкладка

`document.hidden === true`, поэтому **rAF мёртв: 0 кадров в секунду**. Из этого следует:

- Инструмент `computer{action:"screenshot"}` **виснет по тайм-ауту**. Не пытайся.
- Игра не идёт сама — кадры надо крутить руками:

```js
window.__clock = performance.now();
window.pump = (ms) => { const n = Math.round(ms/16.667);
  for (let i=0;i<n;i++){ window.__clock += 16.667; game.loop.step(window.__clock); } };
```
`game.loop.step(t)` принимает время — можно гнать 20 секунд игры мгновенно.
`window.game` выставлен в `main.js` специально для этого.

- **Скриншот снимается так:** `game.renderer.snapshot(cb)` + `pump(34)`, затем
  POST-ом на `/upload?name=x.png` (сервер сохраняет в `tools/_shots/`), и читать
  файл через Read. Гонять base64 через контекст не надо.

### ⚠️ Главная ловушка стенда: твины Phaser живут по РЕАЛЬНЫМ часам

`TweenManager` не берёт дельту из `loop.step()`. Синтетические часы его не двигают.
`pump(6000)` прокручивает 6 секунд игры за ~40 мс реального времени — **твины за это
время почти не сдвигаются**. Из-за этого легко словить «баги», которых нет:
собака масштаба 0.73, котик размером с точку, `body.width = 2` — всё это
недоигравший твин появления.

**Перед замерами и скриншотами гаси твины руками:**
```js
s.tweens.killTweensOf(s.cat); s.cat.setScale(s.cat.baseScale);
s.dogs.getChildren().forEach(d => { s.tweens.killTweensOf(d); d.setScale(d.baseScale).setAlpha(1); });
```
`await sleep()` не спасает: в фоновой вкладке `setTimeout` зажат до 1 секунды.

**Внимание:** если браузерная панель в СЕССИИ активна и не в фоне (не hidden),
rAF наоборот тикает сам по РЕАЛЬНОМУ времени между отдельными вызовами
инструмента — таймстемпы вроде `distractedUntil`/`frozenUntil`, выставленные в
одном вызове, могут уже истечь к следующему. Проверяй длинные сценарии ОДНИМ
непрерывным вызовом, а не разбивая на несколько с паузой между ними.

### Альтернатива: headless Playwright (`tools/pw/`)

Когда браузерной панели вообще нет в сессии (нет `computer`/browser-use
инструмента), можно ставить headless Chromium локально — это НЕ часть игры,
отдельный dev-инструмент, `node_modules` в `.gitignore`.

```
cd tools/pw
npm install                      # один раз
npx playwright install chromium  # один раз, качает браузер (~180 МБ)
npm run shop                     # прогоняет Shop-сцену, шлёт скрины в tools/_shots/
```

`shop-test.mjs` сам поднимает статический сервер на 8080 (свой, на `node:http` —
`python`/`serve.ps1` не нужны), открывает `?nosw`, ждёт `game.scene.isActive('Menu')`
(чтобы ассеты уже были в текстур-кеше), переключает на `Shop` напрямую через
`window.game.scene.start('Shop')`, свайпает мышью и сверяет `scrollY`/`maxScroll`.

**Чем это лучше браузерной панели:** страница не в фоновой вкладке — `document.hidden
=== false`, rAF и твины идут сами по реальному времени, `pump()` не нужен. И
`page.mouse` шлёт настоящие pointer-события — обходной манёвр с синтетическим
`MouseEvent` (см. выше про джойстик) тут не требуется, `pointerdown`/`pointerup`
работают как есть.

⚠️ На этой машине `python`/`python3` в PATH — заглушки Windows Store, а не
настоящий интерпретатор (не верь `--version`, реально не запускается). Node
настоящий и есть — поэтому статический сервер для Playwright написан на нём,
а не на python, в отличие от `tools/serve.ps1`.

### Ещё грабли стенда (не путать с багами игры)

- **Сцена переиспользуется.** `game.scene.getScene('Game')` возвращает ТОТ ЖЕ объект
  после `restart`. Свои флаги для тестов (`s._caught`) **протекают между прогонами** —
  так можно получить ложное «укрытие не работает, 8/8». Держи флаг в локальной
  переменной замыкания, не на сцене.
- **Phaser не слушает `pointerdown`.** Джойстик и кнопки тестируй
  `MouseEvent('mousedown'/'mousemove'/'mouseup')`. С `PointerEvent` всё молча не работает.
- **`game.scale.resize()` откатывается** (режим RESIZE подгоняет канвас под окно).
  Поворот экрана проверяй настоящим `resize_window` + `pump`.
- **`caches.open(undefined)`** создаёт мусорный кеш с именем `"undefined"`. Проверяй,
  что `keys[0]` существует.
- Консоль показывает историю всех загрузок — старые предупреждения ≠ текущее состояние.
  Проверяй факты через JS, а не по логам.
- **Модуль `progress` кеширует `state` в памяти.** Прямой `localStorage.setItem` в
  тесте НЕ обновляет закешированный `state` — тест мерит устаревшее. Чтобы задать
  состояние для проверки: `setItem` + **перезагрузка страницы** (модуль прочитает
  свежее), либо гоняй через сам модуль. `import('...js?v=')` создаёт ВТОРОЙ
  экземпляр с отдельным `state` — не совпадает с тем, что у сцен (импорт без query).

## Баги, которые уже ловили — не верни обратно

- **Собаки были безобидными.** Спавнились в 67 шагах, а за 15 секунд жизни проходят
  ~43 тайла → не добегали НИКОГДА. У спавна был только минимум дистанции, без максимума.
  **Не убирай верхнюю границу спавна `dogReach`** — без неё баг вернётся. Сейчас она
  считается от `DOG_APPROACH_BUDGET`, а 15 секунд погони идут со встречи, поэтому
  окно широкое (7..54/62/70 тайлов) и разброс честно случайный.
- **Утечка слушателей.** Phaser не чистит `this.events` при restart → `dog-vanished`
  копился с каждым перезапуском. Обработчик снимается в `shutdown`. Проверяй, что
  счётчик держится на 1.
- **Коробка могла встать на выход/старт.** На уровне 1 тупиков бывает всего 2, их
  забирают старт с выходом → запасной вариант ставил коробку куда попало.
- **Чёрный экран при старте.** Проверка звуков стояла в `await` без таймаута — сервер
  не ответил, игра не запустилась. Сейчас гонка с таймером: **запуск не должен
  зависеть от сети, никогда**.
- **SW отдавал `index.html` на любой неудачный запрос** → проверка «есть ли mp3»
  получала HTML со статусом 200 и Phaser давился им. Подмена только для `mode === 'navigate'`.
- **Зум камеры не пересчитывался** при повороте телефона (считался раз в `create`).
- **Таймер позы «сесть» не отменялся** и стрелял посреди следующей анимации.
- **Стая уходила вразнобой**: сигнал `dog-vanished` эмитился после 400-мс анимации
  растворения, и чужой таймер успевал дотикать в эту щель. Эмит — сразу в `vanish()`.
- **Замена собаки не спавнилась**: `_spawnDog` считал «умирающих» (dead, но ещё
  играющих анимацию) как живых. Считать только `!d.dead`.
- **Хит-зона кнопки в контейнере**: координаты `Geom.Circle` для `setInteractive` —
  ЛОКАЛЬНЫЕ для объекта, начало в левом верхнем углу рамки. Центр = (радиус, радиус),
  а не (0,0) — иначе зона уезжает и кнопка молчит.
- **Котики были разного размера** (Зефир мельче всех). Две причины сразу:
  1) нарезчик масштабировал по РАМКЕ вырезки, а не по самому котику — пустые поля
     съедали размер (теперь `ContentBox` считает тесную рамку);
  2) **ряды контактного листа нарисованы в разном масштабе** — нижний крупнее
     верхних. Позы ходьбы ОБЯЗАНЫ браться из одного ряда.
  Масштаб теперь ОДИН на весь набор, иначе сидящий котик крупнее идущего,
  а собака пульсирует в размере прямо на бегу.
- **Оттенок шашечки отличается на разных листах**: у `Cat.png` тёмная клетка ~60,
  у `dog_2.png` ~38, у новых листов ~6. Узкий фильтр оставлял чёрные пятна по краям
  спрайтов (и для чёрно-белых зверей flood просачивался сквозь фигуру, разрезая её).
  Решение: фон определяется по УЗОРУ шашечки (пиксель — фон, только если рядом на
  расстоянии клетки есть клетка ПРОТИВОПОЛОЖНОГО тона с ≥2 сторон), а тона и размер
  клетки калибруются по краям каждого листа (`CalibrateBackground`/`IsBackgroundAt`
  в `tools/slicerlib.ps1`).
- **launched-сцена ещё не прошла `create()`.** `GameScene.create` вызывал
  `ui.setCoins()` сразу после `scene.launch('UI')` — а её label ещё null, краш.
  Стартовые данные для HUD передавай в объекте launch (`{ coins }`), а не вызовом.
- **Все бесплатные скины сейчас `price:0`** = всегда открыты. Путь ПОКУПКИ на них
  не проверить (`buyCat` вернёт true, ничего не списав — уже открыт). Тестируй
  покупку на несуществующем платном id (`buyCat('testcat', 20)`) — та же ветка.
- **Замороженная щитом собака ловила ПОВТОРНО в тот же прыжок.** `freeze()` только
  останавливает (`setVelocity(0,0)`), тело физики остаётся включённым и в зоне
  касания. Кадром позже (щит уже снят) overlap срабатывал снова → мгновенный
  нелегитимный проигрыш сразу после «спасения». Фикс: `_onDogTouch` проверяет
  `dog.frozenUntil > time` РАНЬШЕ, чем `cat.isSafe`/`cat.hasShield` — замороженная
  собака не ловит вообще, это не «щит спас», а «оглушённый хищник не опасен».

## Ассеты и правило про кеш

**`images/`** — исходные контактные листы (~85 МБ). Игра их **не грузит**, это
архив. СТАРЫЕ листы (`Cat.png`,`Dog.png`,`dog_2.png`): по 4/2 персонажа на лист.
НОВЫЕ листы (`player_*`,`attack_*`): по ОДНОМУ персонажу на лист, раскладка «6
кадров бега + 2 стойки + 2 позы». ⚠️ У Дымка в нижнем ряду ПРАВАЯ фигура испорчена
водяным знаком — берётся левая. ⚠️ `Select-Object -First N` обрывает конвейер и
убивает нарезной скрипт на полпути.

Вес самой игры (то, что грузит телефон): **1.47 МБ** (Phaser 1.13 МБ + спрайты 261 КБ).

**Поменял файлы для игроков → подними `CACHE_NAME` в `sw.js`** (не верь числу в
этой памятке — открой `sw.js` и посмотри актуальное). Иначе у игроков останется
старая версия. Добавил новый ассет ИЛИ файл сцены/конфига — впиши его в `ASSETS`
в `sw.js`, иначе офлайн его не будет.
