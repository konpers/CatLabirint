# CLAUDE.md — рабочая память по проекту

Игра-лабиринт для детей 9–13 лет. Главный игрок и тестировщик — **Лиана**.
Всё общение и комментарии в коде — **на русском**.

Игра доделана и проверена. Ниже то, что **не восстановить из кода**: почему
сделано так, где грабли и как тут вообще тестировать.

**Сейчас в игре:** 8 уровней, монеты (за прохождение +5 и разбросаны по карте),
магазин со скинами котиков и собак, экран выбора уровня (он же фарм), прогресс
сохраняется в localStorage. **8 игроков** (4 беспл. + 4 платных: Честер/Стрелка/
Барс/Панда) и **7 врагов** (2 беспл. + 5 платных: медведи Охотник/Бурый/Глыба,
Тайсон, Убийца). Все платные покупаются за монеты в магазине. **Бег теперь
единственная скорость** (кнопку-переключатель убрали) — освободившееся место
занимает бар суперспособностей: **косточка** (25 монет, кнопка-активация,
отвлекает собак в радиусе 20 тайлов на 6 сек) и **щит** (35 монет или редкий
пикап на карте, прощает одну поимку, замораживает поймавшую собаку на 4 сек).

## 1. Как запустить и проверить (важно — тут всё нестандартно)

Открыть `index.html` по `file://` нельзя — браузер запретит ES-модули и Service Worker.
Нужен локальный сервер.

**Python 3.14 и Node v24 установлены** (июль 2026). Самое простое:

```
python -m http.server 8080     # из корня проекта
# затем http://localhost:8080/?nosw
```

⚠️ **Но старый процесс их не увидит.** Node ставился при уже запущенной сессии, и
мой PowerShell унаследовал прежний PATH: `node` в нём не находится, хотя лежит в
`C:\Program Files\nodejs\node.exe` и работает по полному пути. Если `node`/`python`
«не найдены» — не верь, проверь полным путём или через реестр:
`[Environment]::GetEnvironmentVariable("Path","Machine")`.

Запасной сервер на PowerShell (зависимостей не требует, умеет принимать скриншоты
POST-ом на `/upload`):

```
powershell -ExecutionPolicy Bypass -File tools\serve.ps1
```

**Всегда открывай с `?nosw`.** Service Worker кеширует всё и отдаёт старый код —
за сессию это дважды съело время на отладку призрачных багов. `?nosw` снимает SW
и чистит кеш.

### Скриншоты и автотесты: браузерная панель — фоновая вкладка

`document.hidden === true`, поэтому **rAF мёртв: 0 кадров в секунду**. Из этого следует:

- Инструмент `computer{action:"screenshot"}` **виснет по тайм-ауту**. Не пытайся.
- Игра не идёт сама — кадры надо крутить руками:

```js
window.__clock = performance.now();
window.pump = (ms) => { const n = Math.round(ms/16.667);
  for (let i=0;i<n;i++){ window.__clock += 16.667; game.loop.step(window.__clock); } };
```
`game.loop.step(t)` принимает время — можно гнать 20 секунд игры мгновенно.
`window.game` выставлен в `main.js` специально для этого.

- **Скриншот снимается так:** `game.renderer.snapshot(cb)` + `pump(34)`, затем
  POST-ом на `/upload?name=x.png` (сервер сохраняет в `tools/_shots/`), и читать
  файл через Read. Гонять base64 через контекст не надо.

### ⚠️ Главная ловушка стенда: твины Phaser живут по РЕАЛЬНЫМ часам

`TweenManager` не берёт дельту из `loop.step()`. Синтетические часы его не двигают.
`pump(6000)` прокручивает 6 секунд игры за ~40 мс реального времени — **твины за это
время почти не сдвигаются**.

Из-за этого я трижды ловил «баги», которых нет: собака масштаба 0.73, котик размером
с точку, `body.width = 2`. Всё это — недоигравший твин появления.

**Перед замерами и скриншотами гаси твины руками:**
```js
s.tweens.killTweensOf(s.cat); s.cat.setScale(s.cat.baseScale);
s.dogs.getChildren().forEach(d => { s.tweens.killTweensOf(d); d.setScale(d.baseScale).setAlpha(1); });
```
`await sleep()` не спасает: в фоновой вкладке `setTimeout` зажат до 1 секунды.

### Ещё грабли стенда (не путать с багами игры)

- **Сцена переиспользуется.** `game.scene.getScene('Game')` возвращает ТОТ ЖЕ объект
  после `restart`. Свои флаги для тестов (`s._caught`) **протекают между прогонами** —
  once я так получил ложное «укрытие не работает, 8/8». Держи флаг в локальной
  переменной замыкания, не на сцене.
- **Phaser не слушает `pointerdown`.** Джойстик тестируй `MouseEvent('mousedown'/'mousemove'/'mouseup')`.
  С `PointerEvent` всё молча не работает.
- **`game.scale.resize()` откатывается** (режим RESIZE подгоняет канвас под окно).
  Поворот экрана проверяй настоящим `resize_window` + `pump`.
- **`caches.open(undefined)`** создаёт мусорный кеш с именем `"undefined"`. Проверяй,
  что `keys[0]` существует.
- Консоль показывает историю всех загрузок — старые предупреждения ≠ текущее состояние.
  Проверяй факты через JS, а не по логам.

## 2. Решения и почему именно так (не менять не подумав)

| Решение | Почему |
|---|---|
| Phaser 3 локально в `vendor/` | С CDN не работает офлайн, а офлайн — требование PWA |
| Вид сверху с наклоном, **не изометрия** | Ромбы хуже читаются на узком экране телефона |
| Стены рисованные, не тайлсет | `images/Lab_walls.png` — **изометрический**, на квадратную сетку не ложится |
| Генератор: recursive backtracker | Гарантирует, что выход достижим всегда (проверено на 1000 лабиринтов) |
| «Плетение» тупиков (`braid`) | Чистый DFS = одни тупики. С 3 собаками некуда отступать — нечестно |
| Поиск пути: BFS, не A* | Карта ≤35×35. BFS даёт кратчайший путь без эвристик; A* тут лишний код |
| Пересчёт маршрута раз в 500 мс | Дешевле и выглядит живее — собака будто думает |
| Собака медленнее бегущего котика (170) | Иначе убежать нельзя в принципе. Доля задаётся уровнем: 0.7/0.8/0.9 → 119/136/153 |
| 15 сек собаки идут **со встречи**, а не со спавна | Правка после теста с Лианой: спавн стал случайным по всей карте, но за 15 сек собака пробегает ~40 тайлов при медиане 65–88 и максимуме 326 — далёкая собака не дошла бы никогда |
| Таймер стаи общий: истёк у одной — уходят ВСЕ | Правка после теста с Лианой: с личными таймерами собаки исчезали/появлялись по кругу без передышки. Сигнал `vanish('chase')` эмитится ДО анимации растворения, иначе стая уходит вразнобой. Возвращаются волной через respawnGap |
| Кнопка «БЕГ» удалена, бег теперь ЕДИНСТВЕННАЯ скорость | Сначала была переключателем (жалоба с теста на неудобство держать её и джойстик одной рукой), потом пользователь попросил убрать вовсе — котик всегда на `SPEED.catRun`. `catWalk` и весь `input.running` (Cat.js, Joystick.js) выпилены как мёртвый код. Освободившееся место в HUD занял бар способностей |
| Порода собаки случайна при каждом спавне | Стая разношёрстная, встречи разнообразнее. Породы — в `src/config/dogs.js`, добавление новой описано там же |
| Пауза ⏸ (правый верх) → Продолжить / Заново / В меню | Просьба с теста. «Заново» = restart текущего уровня (лабиринт будет новый — так устроен restart) |
| `roundPixels: true` в конфиге рендера | Котик «размазывался» на телефоне: при дробном зуме камеры (~0.87) спрайт рисовался между пикселями |
| Пол рисуется одним `Graphics` | Тысяча спрайтов = 20 fps на 5-м уровне вместо 60 |
| Весь прогресс — 1 JSON в localStorage (`cat_maze_save`) | В подобных играх сохранение = локальное хранилище браузера, сервер не нужен. Всё через `src/config/progress.js`, формат в одном месте. Миграция старого `cat_maze_skin` → `selectedCat` |
| Монеты банкуются ТОЛЬКО при победе | Иначе фарм = ферма смертей: зашёл, собрал, умер, повтор. Собранное в забеге живёт в `GameScene.collected`, коммитится в `_win`, при `_lose` сгорает |
| Скины data-driven: `price` в cats.js/dogs.js | `price:0` = бесплатный, открыт всегда (форс-разблокировка в progress). Новый платный скин — строка с `price>0` + картинки; магазин подхватит |
| Игрок сам включает породы врагов (enabledDogs) | Купил породу — включена; `randomEnemyBreed` тянет только из enabled; минимум одна активна всегда |
| После 8-го уровня → LevelSelect, не Menu | Чтобы фарм был под рукой: перепройти любой открытый уровень |
| Способности — стек зарядов (`progress.items`), НЕ разовая разблокировка | Купил дважды — 2 использования. Отдельный generic API (`getItemCount/buyItemCharge/useItemCharge`) от `unlockedCats`-паттерна: там «куплено раз и навсегда», тут расходуется |
| Щит — БЕЗ кнопки, применяется автоматически | Куплен в магазине → заряд списывается и аура включается на старте СЛЕДУЮЩЕГО уровня. Найден на карте → применяется мгновенно при подборе. Один активный тип кнопки (для косточки) проще для ребёнка, чем две |
| Радиус косточки — BFS-шаги по лабиринту, не по прямой | `distanceMap` от тайла косточки — та же утилита, что уже считает дальность спавна собак. По прямой через стену собака бы «слышала» косточку нечестно |
| `BONE_DISTRACT_MS` считается от АКТИВАЦИИ, а не от прихода к косточке | ТЗ описывало «5 сек стоят у косточки», но отслеживать приход каждой собаки отдельно — лишняя сложность. Общий таймер на 6 сек (запас на подход) визуально неотличим |
| Замороженная собака (`frozenUntil`) не ловит НИКОГДА, не только пока щит активен | Поймано на тестах: собака стоит на месте, физически остаётся в зоне касания и на следующем кадре (щит уже снят) тут же засчитывала вторую нелегитимную поимку тем же прыжком. Простая проверка `dog.frozenUntil > time` в `_onDogTouch` — до проверки `hasShield` |

**Правило читаемости:** стена обязана быть заметно **темнее** пола. Первая версия
провалилась именно на этом — светлый пол и светлая стена сливались, было непонятно,
где проход. Стены собираются в блоки тёмным контуром + боковая грань снизу (эффект 2.5D).

## 3. Баги, которые уже ловили — не верни обратно

- **Собаки были безобидными.** Спавнились в 67 шагах, а за 15 секунд жизни проходят
  ~43 тайла → не добегали НИКОГДА. У спавна был только минимум дистанции, без максимума.
  **Не убирай верхнюю границу спавна `dogReach`** — без неё баг вернётся. Сейчас она
  считается от `DOG_APPROACH_BUDGET`, а 15 секунд погони идут со встречи, поэтому
  окно широкое (7..54/62/70 тайлов) и разброс честно случайный.
  Тест: ловит стоящего котика 5/6, 6/6, 6/6 (уровни 3/4/5); время до встречи 6–25 сек.
- **Утечка слушателей.** Phaser не чистит `this.events` при restart → `dog-vanished`
  копился с каждым перезапуском. Обработчик снимается в `shutdown`. Проверяй, что
  счётчик держится на 1.
- **Коробка могла встать на выход/старт.** На уровне 1 тупиков бывает всего 2, их
  забирают старт с выходом → запасной вариант ставил коробку куда попало.
- **Чёрный экран при старте.** Проверка звуков стояла в `await` без таймаута — сервер
  не ответил, игра не запустилась. Сейчас гонка с таймером: **запуск не должен
  зависеть от сети, никогда**.
- **SW отдавал `index.html` на любой неудачный запрос** → проверка «есть ли mp3»
  получала HTML со статусом 200 и Phaser давился им. Подмена только для `mode === 'navigate'`.
- **Зум камеры не пересчитывался** при повороте телефона (считался раз в `create`).
- **Таймер позы «сесть» не отменялся** и стрелял посреди следующей анимации.
- **Стая уходила вразнобой**: сигнал `dog-vanished` эмитился после 400-мс анимации
  растворения, и чужой таймер успевал дотикать в эту щель. Эмит — сразу в `vanish()`.
- **Замена собаки не спавнилась**: `_spawnDog` считал «умирающих» (dead, но ещё
  играющих анимацию) как живых. Считать только `!d.dead`.
- **Хит-зона кнопки в контейнере**: координаты `Geom.Circle` для `setInteractive` —
  ЛОКАЛЬНЫЕ для объекта, начало в левом верхнем углу рамки. Центр = (радиус, радиус),
  а не (0,0) — иначе зона уезжает и кнопка молчит.
- **Котики были разного размера** (Зефир мельче всех). Две причины сразу:
  1) нарезчик масштабировал по РАМКЕ вырезки, а не по самому котику — пустые поля
     съедали размер (теперь `ContentBox` считает тесную рамку);
  2) **ряды контактного листа нарисованы в разном масштабе** — нижний крупнее
     верхних. Позы ходьбы ОБЯЗАНЫ браться из одного ряда.
  Масштаб теперь ОДИН на весь набор, иначе сидящий котик крупнее идущего,
  а собака пульсирует в размере прямо на бегу.
- **Оттенок шашечки отличается на разных листах**: у `Cat.png` тёмная клетка ~60,
  у `dog_2.png` ~38. Узкий фильтр оставлял чёрные пятна по краям спрайтов.
  Сейчас диапазон 28..135 (см. `IsChecker` в `tools/slicerlib.ps1`).
- **launched-сцена ещё не прошла `create()`.** `GameScene.create` вызывал
  `ui.setCoins()` сразу после `scene.launch('UI')` — а её label ещё null, краш.
  Стартовые данные для HUD передавай в объекте launch (`{ coins }`), а не вызовом.
- **Модуль progress кеширует state в памяти.** Прямой `localStorage.setItem` в тесте
  НЕ обновляет закешированный `state` — тест мерит устаревшее. Чтобы задать состояние
  для проверки: `setItem` + перезагрузка страницы (модуль прочитает свежее), либо
  гоняй через сам модуль. `import('...js?v=')` создаёт ВТОРОЙ экземпляр с отдельным
  state — не совпадает с тем, что у сцен (импорт без query).
- **Все 4 котика и 2 собаки сейчас `price:0`** = бесплатны и всегда открыты. Путь
  ПОКУПКИ на них не проверить (buyCat вернёт true, ничего не списав — уже открыт).
  Тестируй покупку на несуществующем платном id (`buyCat('testcat', 20)`) — та же ветка.
- **Замороженная щитом собака ловила ПОВТОРНО в тот же прыжок.** `freeze()` только
  останавливает (`setVelocity(0,0)`), тело физики остаётся включённым и в зоне
  касания. Кадром позже (щит уже снят) overlap срабатывал снова → мгновенный
  нелегитимный проигрыш сразу после «спасения». Фикс: `_onDogTouch` проверяет
  `dog.frozenUntil > time` РАНЬШЕ, чем `cat.isSafe`/`cat.hasShield` — замороженная
  собака не ловит вообще, это не «щит спас», а «оглушённый хищник не опасен».

## 4. Устройство

```
src/config/   assets.js (картинки + PALETTE), levels.js (8 уровней + монеты + баланс
              способностей), cats.js (скины котиков), dogs.js (породы собак),
              abilities.js (косточка/щит: цена, kind active/passive),
              progress.js (СОХРАНЕНИЕ: монеты/скины/прогресс/инвентарь, 1 ключ localStorage)
src/scenes/   Boot → Menu → LevelSelect → Game + UI; Shop (магазин/кастомизация)
src/systems/  MazeGenerator, Pathfinder (BFS), Joystick (плавающий, левая половина)
src/entities/ Cat (движение + 5 idle + hasShield), Dog (погоня + distract/freeze)
tools/        нарезка спрайтов из контактных листов + локальный сервер
```

Навигация сцен: Menu (выбор котика, монеты, магазин) → LevelSelect (8 узлов, гейт по
`maxLevel`, фарм) → Game (+UI HUD). Shop открывается из Menu, LevelSelect и паузы.
Прохождение ведёт на следующий уровень; последний — в LevelSelect.

- **Способности:** `src/config/abilities.js` — data-driven, как cats.js/dogs.js.
  `kind: 'active'` (косточка) получает кнопку в HUD (UIScene `_makeAbilityBar`,
  цикл готов к 3-4 будущим способностям без переделки); `kind: 'passive'` (щит)
  кнопки не получает — применяется в GameScene автоматически. Инвентарь — стек
  зарядов в `progress.items`, НЕ разовая разблокировка (см. таблицу решений).

- **Ассеты:** все пути к картинкам — только в `src/config/assets.js`, больше нигде.
  Персонаж на картинке смотрит **вправо** (влево игра отражает сама). Размер на экране
  задают `CAT_SIZE`/`DOG_SIZE` в `levels.js`, а не размер картинки.
- **Спрайты:** 8 персонажей-игроков × 3 позы (`cat_<id>_walk/sit/sleep`) — 4
  бесплатных котика + 4 платных (`bengal`,`cornish`,`mainecoon`,`kolli`; последний —
  собака-игрок, играется через тот же слот котика). Враги — 7 пород
  (`beagle`,`bull` + `bear_black`,`bear_brown`,`bear_white`,`pitbull`,`shepherd`)
  × 6 кадров бега + `wait`. Коробка/домик/монета — рисованные SVG.
- **`images/`** — исходные контактные листы (~85 МБ). Игра их **не грузит**, это архив.
  СТАРЫЕ листы (`Cat.png`,`Dog.png`,`dog_2.png`): по 4/2 персонажа на лист,
  нарезка `tools/cut_cats.ps1`/`cut_dogs.ps1`. НОВЫЕ листы (`player_*`,`attack_*`):
  по ОДНОМУ персонажу на лист, раскладка «6 бега + 2 стойки + 2 позы», нарезка
  `tools/cut_new.ps1` (зоны ячеек + ContentBox).
  `tools/preview.ps1 <лист> <минПлощадь> <выход>` рисует найденные рамки с номерами.
  ⚠️ У Дымка в нижнем ряду ПРАВАЯ фигура испорчена водяным знаком — берётся левая.
  ⚠️ `Select-Object -First N` обрывает конвейер и убивает скрипт на полпути.

### ⚠️ Нарезка (slicerlib.ps1): фон определяется по УЗОРУ, не по цвету

Чёрно-белые персонажи (колли, овчарка, чёрный медведь) на шашечном фоне: чёрный
мех ≈ тёмная клетка, серые тени меха ≈ светлая клетка. Простым порогом яркости
не отделить — flood просачивался СКВОЗЬ фигуру, разрезая её.
Решение (`IsBackgroundAt`): пиксель = фон, только если это регулярный УЗОР —
рядом (на расстоянии клетки, ±4px по диапазону) есть клетка ПРОТИВОПОЛОЖНОГО тона
с ≥2 сторон. Внутри сплошного меха узора нет; граница чёрн/бел мех даёт «другой
тон» лишь с 1 стороны, поэтому не выедается.
Тоны клеток и размер клетки (`Cell`) КАЛИБРУЮТСЯ по краям каждого листа
(`CalibrateBackground`) — у `Cat.png` тёмная клетка ~60, у новых ~6, клетка везде ~23px.
`CleanupBackground` дочищает тонкие остаточные линии антиалиаса.
Тёмные враги дают лёгкие остатки шашечки, но в игре (48px) незаметны.
- **Вес игры: 1.47 МБ** (Phaser 1.13 МБ + спрайты 261 КБ).
